## Key Decisions

1. **Separate skill for profiles** — `/build-stakeholder-profile` is standalone, not Phase -1 inside ds-brainstorm. Reason: context budget (profile building = 30-40% context, would blow brainstorm budget)
2. **Parallel-first dispatch** — Default parallel, auto-degrade to sequential. No config needed.
3. **Graceful degradation everywhere** — Never block. User-context-only brainstorm is still valuable.
4. **Web search as PoC data source** — Confluence MCP available internally but not locally. Architecture is source-agnostic.
5. **Agent-as-judge evals** — Quality rubric scored by the coding agent inline, no external API needed.
